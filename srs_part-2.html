Software Requirements Specification (SRS) - Exam Web App

1. Introduction

1.1 Purpose

The purpose of this document is to specify the requirements for the development of an exam web app that allows users to create cohorts, conduct assessments, and manage exam-related activities. This is a whole new system that is being created for taking exams with proper security. This is a new system that is not any patch for any other system. We will also create a mobile app for the convenience for the audience.

1.3 Intended Audience and Reading Suggestions

This document is intended for developers, project managers, faculty members, students, testers, and documentation writers. It is organized to provide an overview of the app's functionality, features, and user interactions.

1.4 Product Scope

This Exam Taking Web Application is a user-centric platform designed for taking the exams and assessment of the exams smoothly. It allows students to take exams online, providing instant feedback and enabling remote participation. Teachers benefit from easy to create question papers functionality, automated grading, live feedback of the students while taking exams, and revise the grades later if needed. The application aligns with educational institutions' goals of efficiency, accessibility, and academic excellence, offering a secure and innovative solution for modernizing the exam experience.

2. Overall Description

2.1 Product Perspective

The Exam Taking Web Application is being developed as a standalone product, developed to make the exam taking procedure easy, secure and transparent. It is not part of an existing product family but serves as a self-contained solution to modernize and optimize the exam administration and assessment procedures.

2.2 Product Functions

  • User registration and sign-up through email.
  • Creation of cohorts for taking exams for a certain selected students.
  • Faculty-initiated invitation links for students to join cohorts.
  • Ability to add multiple-choice, text, and speech-based questions.
  • Question paper availability within specified time windows.
  • Automatic question paper expiration after a set duration.
  • Faculty-provided answer keys for assessment evaluation.
  • Automatic marking and calculation of student marks.
  • Display of marks to faculty and students.
  • Student dashboard for queries and mark-related complaints.
  • Image-based student verification using 3D images.
  • Microphone and video feedback while taking exams for the transparency of the exam.
  • Automatic reporting of any kind of cheating during the exam.
  • Students cannot change tab while taking exams due to security purpose.

2.3 User Classes and Characteristics

  • Faculty: Creates assessments, manages cohorts, and evaluates student answers, monitor exam sessions, provide answers, update marks.
  • Students: Participate in assessments, submit complaints, and view results.

2.4 Operating Environment

The web app operates in modern web browsers and requires an internet connection. It should be accessible across different devices. For mobile devices, there will be mobile application.

Hardware Platform:

Desktop and laptop computers need to have at least 4GB of RAM and a modern processor. And mobile devices(Android) need to have enough sufficient memory and a decent processor to run the mobile application.

Web Browsers:

  • Google Chrome(latest version)
  • Mozilla Firefox (latest version)
  • Microsoft Edge (latest version)

Software Dependencies:

  • Web Server: The application requires a web server, such as Apache or Nginx, to serve web pages and manage user requests.
  • Backend Framework: The application is built using a specific backend framework (e.g., Django, Ruby on Rails), which handles data processing, authentication, and interactions with the database.
  • Database Management System: The application uses a relational database management system (e.g., MySQL, PostgreSQL, Firebase) to store user data, assessment details, and results.
  • Image Processing Libraries: To verify student identity, the application interfaces with image processing libraries such as OpenCV.
  • External Libraries: There maybe many external libraries to make the functionality of the app smooth.

External Services

Email Services: The application integrates with third-party email services (e.g., SendGrid, SMTP servers) to send invitation links and notifications to users.

2.5 Design and Implementation Constraints

  • Use of secure authentication and authorization mechanisms.
  • Implementation adheres to privacy and data protection regulations.
  • Integration with camera and microphone for image verification.

2.6 User Documentation

The app will provide user documentation including user guides, FAQs, and tutorials.

2.7 Assumptions and Dependencies

  • Availability of stable internet connections.
  • Users have access to modern web browsers.
  • Image verification is feasible using 3D images.
